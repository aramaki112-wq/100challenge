# DAY14 thought-process.md

# Todoリストアプリをどう設計する？

---

## 💭まず最初に考える

Todoアプリって何だろう？

やることを追加して

一覧で見られて

不要になったら削除する。

すごくシンプルなアプリだ。

でも、

プログラムとして考えると

最初に決めることがある。

---

## 💭Todoはどこへ保存する？

画面？

いや違う。

画面は表示するだけ。

データそのものを保存する場所が必要。

変数かな？

```

```javascript
let todo = "";
```

```
いや、

Todoは1個じゃない。

10個になるかもしれない。

100個になるかもしれない。

変数では足りない。

---

## 💭じゃあ配列かな

配列なら

```

```
[]

↓

["勉強"]

↓

["勉強","散歩"]

↓

["勉強","散歩","買い物"]
```

```
自由に増やせる。

減らすこともできる。

今回必要なのはこれだ。

```

```javascript
const todos = [];
```

```
これをアプリの本体にしよう。

---

## 💭じゃあ画面は？

画面には

Todo一覧を表示したい。

最初に思いつくのは

「追加されたらHTMLを追加する」

という方法。

例えば

```

```javascript
todoList.innerHTML += ...
```

```
これでも作れる。

でも少し考えてみる。

---

## 💭削除するとどうなる？

Todoが

```

```
勉強

散歩

買い物
```

```
あったとする。

真ん中だけ消したい。

HTMLを直接触ると

かなり面倒そう。

どの要素？

どの位置？

兄弟要素は？

順番は？

考えることが増える。

---

## 💭もっと簡単な方法はないかな？

配列だけ正しくすれば

画面は作り直してもいいんじゃない？

例えば

```

```
配列

↓

全部表示

↓

完成
```

```
追加しても

削除しても

最後に

全部描き直せばいい。

これなら

毎回同じ処理になる。

---

## 💭renderという関数を作ろう

画面を描く処理は

何回も使う。

追加した時

削除した時

あと将来

保存データを読み込んだ時も

使える。

だから

```

```javascript
render();
```

```
という関数にまとめよう。

---

## 💭renderの役目は1つだけ

renderは

「画面を作るだけ」

追加しない。

削除しない。

保存もしない。

表示だけ。

役割を分けると

後で読みやすくなる。

---

## 💭最初に画面を空にする理由

renderの最初で

```

```javascript
todoList.innerHTML = "";
```

```
としている。

最初は

「全部消すなんてもったいない」

と思った。

でも

全部消して

全部作り直した方が

実はシンプルだった。

コードも短い。

バグも減る。

初心者にも分かりやすい。

---

## 💭Todoを1件ずつ表示しよう

配列には

何件入るか分からない。

だから

1件ずつ取り出す必要がある。

for文でも書ける。

でも今回は

```

```javascript
forEach()
```

```
を使おう。

配列を順番に回す、

という目的が分かりやすい。

---

## 💭削除はHTMLじゃなく配列

削除ボタンを押したら

最初は

liを消そうと思った。

でも、

本当に消したいのは

画面じゃない。

データだ。

画面は

データを映しているだけ。

だから

```

```javascript
todos.splice(index,1);
```

```
だけでいい。

そのあと

render()

画面は勝手に最新になる。

---

## 💭追加も同じ考え

追加ボタンを押したら

画面へHTMLを追加する。

最初はそう考えた。

でも

削除と考え方が変わってしまう。

だから

追加も

```

```javascript
todos.push(text);
```

```
だけにする。

最後に

```

```javascript
render();
```

```
これで

追加も削除も

同じ流れになった。

---

## 💭件数表示も配列を見るだけ

Todoが何件あるか。

HTMLを数える？

いや、

配列を見れば分かる。

```

```javascript
todos.length
```

```
これだけでいい。

データが正しいなら

画面も正しい。

---

## 💭Enterキー対応

追加ボタンだけでも十分。

でも

毎回マウスへ手を移動するのは

少し面倒。

Enterでも

addTodo()

を呼べばいい。

処理を関数にまとめておいて

良かった。

同じコードを

2回書かずに済んだ。

---

## 💭今回一番伝えたいこと

このアプリは

Todoアプリではない。

「データと画面を分ける」

という考え方を学ぶための教材だ。

画面を書き換えるのではなく

データを書き換える。

画面は

その結果として更新される。

この考え方は

この先作る

メモ帳

家計簿

日記

在庫管理

生産管理

デジタルツイン

すべて同じになる。

だから

今日覚えるべきことは

pushでも

spliceでもない。

一番大切なのは

```

```
データ

↓

render()

↓

画面
```

```
という設計そのものだ。
